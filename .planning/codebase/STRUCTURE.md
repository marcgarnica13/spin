# Codebase Structure

**Analysis Date:** 2026-03-30

## Directory Layout

```
spin/
├── bin/                    # Executable entry point
│   └── spin               # Main CLI command (installed to /usr/local/bin)
├── lib/                   # Shared library modules
│   ├── spin-common.sh     # Constants and utilities
│   ├── spin-claude.sh     # Session launcher
│   └── spin-status.sh     # Status monitor
├── .planning/             # Planning directory (created by GSD orchestrator)
├── .claude/               # Claude Code configuration and hooks
├── Makefile               # Installation targets
├── README.md              # User documentation
├── LICENSE                # MIT license
├── CHANGELOG.md           # Version history
└── install.sh             # One-liner install script
```

## Directory Purposes

**`bin/`:**
- Purpose: Contains executable CLI entry point
- Contains: Bash script (symlink-safe)
- Key files: `bin/spin`

**`lib/`:**
- Purpose: Contains reusable library modules sourced by main script
- Contains: Modular bash libraries for different features
- Key files: `lib/spin-common.sh`, `lib/spin-claude.sh`, `lib/spin-status.sh`

**`.planning/`:**
- Purpose: GSD orchestrator workspace for planning and codebase documentation
- Contains: Generated documentation and task planning
- Key files: `.planning/codebase/` (ARCHITECTURE.md, STRUCTURE.md, etc.)

**`.claude/`:**
- Purpose: Claude Code integration and configuration
- Contains: Hooks, manifest, settings
- Key files: `.claude/gsd-file-manifest.json`, `.claude/hooks/`, `.claude/settings.json`

## Key File Locations

**Entry Points:**
- `bin/spin`: Main CLI command, handles argument parsing and subcommand dispatch

**Configuration:**
- `Makefile`: Install/uninstall targets
- `.claude/settings.json`: Claude Code session settings
- `.claude/get-shit-done/templates/config.json`: GSD orchestrator config

**Core Logic:**
- `lib/spin-common.sh`: Shared constants (version, session prefix, colors, icons, tree drawing)
- `lib/spin-claude.sh`: `spin_claude()` function for launching sessions
- `lib/spin-status.sh`: `spin_status()` and `detect_claude_state()` for monitoring

**Documentation:**
- `README.md`: User guide with features, requirements, installation, usage
- `CHANGELOG.md`: Version history
- `LICENSE`: MIT license text
- `install.sh`: Installation one-liner

## Naming Conventions

**Files:**
- Entry point: `bin/spin` (no extension)
- Libraries: `lib/spin-<feature>.sh` (descriptive name with feature suffix)
- Scripts: No extension, shebang line `#!/usr/bin/env bash`

**Functions:**
- Module functions: `spin_<function>()` (global namespace under spin prefix)
- Internal functions: No special prefix, but generally start with `spin_`

**Variables:**
- Constants: `UPPERCASE_WITH_UNDERSCORES` (e.g., `SPIN_VERSION`, `SPIN_SESSION_PREFIX`)
- Configuration: `UPPERCASE_WITH_UNDERSCORES` (e.g., `RED`, `GREEN`, `ICON_WORKING`)
- Local variables: `lowercase_with_underscores` (e.g., `session_count`, `project_dir`)

**Environment Variables:**
- Stored in tmux: `SPIN_CWD` (project working directory)
- Passed to Claude: `--worktree <name>` (session-specific workspace)

## Where to Add New Code

**New Feature:**
- Primary code: `lib/spin-<feature>.sh` (new library module)
- Entry point routing: Add case statement in `bin/spin` to source and call new function
- Utilities: Add helper functions to `lib/spin-common.sh` if shared

**New Command:**
- Create: `lib/spin-<command>.sh` with `spin_<command>()` function
- Register: Add case in `bin/spin` around line 25-48
- Document: Update `README.md` with usage and features

**Utilities and Helpers:**
- Shared error/output: `lib/spin-common.sh` (functions: `spin_die()`, `spin_warn()`)
- Colors/icons: Update color/icon constants in `lib/spin-common.sh`
- Tree drawing: Extend constants in `lib/spin-common.sh` (`TREE_BRANCH`, `TREE_LAST`, `TREE_PIPE`)

**`.git/`:**
- Purpose: Git repository metadata
- Generated: Yes (by `git init`)
- Committed: No (hidden directory)

## Installation Structure

**User Installation:**
When installed via `make install` or `install.sh`:
- Binary: `$PREFIX/bin/spin` (default: `/usr/local/bin/spin`, symlink-safe)
- Libraries: `$PREFIX/lib/spin/*.sh` (default: `/usr/local/lib/spin/`)

**Development:**
Run directly from source:
- `./bin/spin` or `/home/marc/Development/spin/bin/spin`
- Libraries auto-sourced from `lib/` relative to script location
